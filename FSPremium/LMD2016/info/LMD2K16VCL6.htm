C++ Builder 6.0 Note
 


Introduction
Since LMD 2009 May releases Delphi/C++ Builder 6.0 support was re-enabled again. For emulating theme features of Delphi 7 and higher parts of ThemeManager package by Mike Lischke (www.soft-gems.net)were integrated into Delphi/C++Builder 6 packages.
[Top]
 

Special Notes

  • Please note that problems may occur, when ThemeManager is installed at the same time.
  • Not all VCL 7 and better features are integrated, e.g. reviewing demos you might note the non-working ParentBackground property for certain controls. Solution is to reset this property.
  • Demos were not adapted/tested for Delphi/C++ Builder 6 support. They usually work, but sometimes you might get messages like "unit masks.dcu not found". Here please update the list of integrated packages or do not use packages at all. Another issue may be usage of controls in demos which are available from Delphi 7 on (example is e.g. XPMan). Here simply ignore error messages during form load. You can emulate this feature by using LMDXPRes unit like in LMD 2007 releases or earlier.

C++ Builder 6.0 Note
You might get problems with the uxTheme.h file originally shipped by Borland during compilation of a project with LMD controls. Error message will be:

  • [C++  Error] uxTheme.h(176): E2146 Need an identifier to declare  (also in line 546)

Simplest solution is to change declarations of the offending types as follows (changed parts marked in bold red):

typedef enum THEMESIZE
{
TS_MIN, // minimum size
TS_TRUE, // size without stretching
TS_DRAW, // size that theme mgr will use to draw part
} myThemeSize;

typedef enum PROPERTYORIGIN
{
PO_STATE, // property was found in the state section
PO_PART, // property was found in the part section
PO_CLASS, // property was found in the class section
PO_GLOBAL, // property was found in [globals] section
PO_NOTFOUND // property was not found
} myPropertyOrigin;
