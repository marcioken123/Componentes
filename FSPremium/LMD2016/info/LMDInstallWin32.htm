Package naming conventions
If you did not choose automatic DPL/BPL installation, you have to install the packages manually. This section describes naming conventions for LMD Package files: 

Runtime package names are constructed as follows: lmdrt[NAME][DB]XX

[Name] is identifier for package name, e.g. lmdsys for LMD SysPack.
[DB] is added for database sensitive package, e.g. lmdrtdbrt160.bpl.

Designtime package names are constructed as follows: dcllmd[NAME][DB]XX
[Name] is identifier for package name, e.g. dcllmdsys160.bpl for LMD SysPack.
[DB] is added for database sensitive package, e.g. dcllmdeldb160.bpl.

XX in package names represents the supported VCL version. Value range starts from 60 (Delphi 6). Following values are possible:
60, 70, 90: Delphi/C++Builder 6, Delphi 7 and 2005
100: BDS 2006 (Delphi / C++ personality)
110: CodeGear RAD Studio 2007 (Delphi / C++ personality)
120: CodeGear RAD Studio 2009 (Delphi / C++ personality)
140: Embarcadero RAD Studio 2010 (Delphi / C++ personality)
150: Embarcadero RAD Studio XE (Delphi / C++ personality)
160: Embarcadero RAD Studio XE2 (Delphi / C++ personality)

Example:
lmdsys11rt_70.bpl  represents LMD SysPack runtime package for Delphi 7.
dcllmdsys10_110.bpl represents LMD SysPack designtime package for Delphi/C++ Builder 2007.

In following Packages overview project groups have different platform code (YY).
Delphi:     D6 represents Delphi 6,  D7 represents Delphi 7.
BDS:          D9-D16 represents Delphi 2005 up to Delphi 2010 and Delphi XE, XE2.
               B10-B16 represents C++ Builder 2006 up to C++ Builder 2010 and C++ Builder XE, XE2.

Example:
_LMDBarPack_d7.bpg represents LMD BarPack project group for Delphi 7.

Source Installation
 Installation of sourcecode was simplified for LMD VCL 2016 packages: For each package project groups as listed above are available. As a rule: For runtime packages press Compile button, for Designtime packages in addition Install button.  Make sure that library path settings reference the source path.

Changing compiler switches
Unlike in previous releases only one compiler switch file exists which is shared by all LMD Packages. It is named lmdcmps.inc. To enable e.g. DebugMode in LMD-Tools, simply enable the LMDDEBUG conditional define.

RAD Studio XE3 (Delphi/C++)

How to add the components
The installation program tries to install the packages automatically (unless you unchecked this option). For installing packages manually, you have to install the LMD VCL designtime packages (check the package list above for available options), Please note, that all designtime package names start with dcl[...] prefix - installing a runtime package does not work. 

Installation

  1. Run Delphi and select "Components" menu. Click on " Install Packages..." button.
  2. Click on the "Add..." button. A dialog box appears. Click there on the "Browse..." button.
  3. From the file dialog select the library path of package(s) you want to install (usually the \lib\d17 directory below your LMD VCL installation folder). If you installed designtime packages during installation to default BPL output folder, please search at that location). 
  4. Select desired designtime package files as mentioned in the package list above. Note: Hold Ctrl-key to select more than one file at the same time.

Important:
If installer did not add path settings automatically, you have to add the full paths (e.g. C:\LMD2016\lib\d17) of LMD VCL DCU files yourself to the Delphi's library path (Library Page/Environment Options in the Tools Menu). 

Important for C++ user:
Because of the fact that no *.obj files are included in precompiled packages, you have to add required *.lib files to your project. For example to add non-db core controls add lmdtools160.lib, for db-controls lmdtoolsdb160.lib. For other sub packages add corresponding runtime lib files as described above.

How to add helpfile into the Delphi-IDE
By default the installer tries to install helpfile automatically. To install any LMD VCL Helpfile manually, check the \help\bds10 folder below installation directory.  All available documentation files for corresponding IDE and the RegHelp2.exe tool can be found here.
To install all available files, simply run RegHelp2. To uninstall all available files, switch to command line and run RegHelp2 /u. To install/uninstall certain files, simply mention the corresponding hxx file. E.g. to uninstall the shared helpfile only, use:
RegHelp2 /u lmdsharedBds10.hxx

Uninstalling LMD VCL packages

To remove LMD VCL packages from your system completely follow the directions below.

Delphi 6, 7 and C++Builder 6

  • Close all files and projects and and select "Components|Install Packages...".
  • Select all LMD VCL packages from the Packages list and click the "Remove" button.
  • After that remove all LMD VCL paths from the global library search path (Library Page/Environment Options in the Tools Menu).
  • Start OpenHelp tool and remove any LMD VCL helpfiles (Menu Help|Customize..., select Tab "Link" then).

Delphi 2005 and higher

  • Close all files and projects and and select "Components|Install Packages...".
  • Select all LMD VCL packages from the Packages list and click the "Remove" button.
  • After that remove all LMD VCL paths from the global library search path (Options|Environment Options in the Tools Menu. See either ).
  • To uninstall helpfile from the IDE check the \help\bdsXXXW folder below installation directory and run RegHelp2 /u from command line (XXXW specifies the BDS version, example is BDS4) to remove the particular help files from IDE. This is only required when automatic installer failed to remove helpfiles.

Removing LMD VCL files completely from your hard disk
After uninstalling the components from the CBuilder IDE/Delphi you may remove the component files physically.

  • Open the Add/Remove Programs icon from the Control Panel and double click the "LMD [..]" entry.
  • All the DCUs/object and backup files have to be removed manually from the LMD VCL' uninstallation directory. As a rule the installation directory won't be deleted during the uninstallation process, because such files will be generated while you work on the Tools but - for safety's sake - won't be deleted automatically by the uninstaller. Alternative: Use the Custom Uninstall method in Uninstaller application and select always all options - this will remove complete paths including generated files. We recommend to use the second way.
  • Remove all dcllmd*.* and lmd*.* files from the \projects\bpl (and  \projects\lib in CBuilder) folders and below your CBuilder/Delphi installation folder. If you had chosen to install runtime files into system directory, remove all lmd*.bpl packages from the System directory of your Windows installation. To be sure search harddisk for these files and remove them all. Do not remove license files or registration utility from the system directory.
  • Be sure that you did not compile LMD VCL files into a custom package!
  • Possible registry keys can be found under:
    HKEY_LOCAL_MACHINE\SOFTWARE\LMD Innovative\LMD-Tools 2011
    Remove them if you don't want to use LMD VCL packages any longer.
