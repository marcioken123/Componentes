 Using different Language Resource File
If available, translated Resourcestrings can be found in

  • \Resources folder below installation directory for compiled versions and
  • \lib\source\strings folder for source code versions

To change current language simply copy unit file of desired language to library file and recompile your application.
Feel free to send us modified resource units for other languages!

DockingPack Info
Welcome to LMD DockingPack! This provides you 100% VCL native controls that can be used to build applications with entirely customizable and modern user interface. New in version 2016 is the Visual Style editor - for more info read the online documentation.

The library has following main features:

  • Advanced site control with horizontal, vertical or tabs dock zones, splitters and shifting "auto hide" zones
  • Transparent hot-spots drag-over active areas.
  • Additional drag-over active areas in tab zones and splitters.
  • Ability to place required user interface controls on the panels (TLMDDockPanel) - prevents having many distinct forms in the project.
  • Advanced design-time features with drag-and-drop zones rearrangement.
  • True proportional zones resizing. The LMD-DockingPack uses double coefficients to store proportional sizes that prevent proportion-lost bug while very slow resizing.
  • The notion of "space" zone. That is the central zone with main application content. The central zone has following useful special properties: space zone have highest priority to resize when the dock site is resizing, leaving all other edge zones with the same size; edge zones "auto hide" side is automatically detected relatively to space zone.
  • Using space zone gives following additional features: Unlike Delphi standard sites, like TPanel, dock-client can be docked into empty site to the edge, not only to the whole site area. This is more naturally from the user viewpoint. Dock-out of the space panel from the site does not corrupt edge zones layout.
  • Allowing locking some panels to prevent them to be dock-out from the site, using DragMode, ShowHeader and Buttons properties of the TLMDDockPanel. Allowing intercepting dock-dropping using OnCustomInsertQuery and OnCustomInsert events of the TLMDDockSite. These two features allow organizing fully custom central zone, like "tabbed documents" in Microsoft Visual Studio.
  • Advanced global saving/loading of the current dock layout. The TLMDDockPersister control allows to save/load any all dock sites, including dynamically created floating sites, in XML format. TLMDDockSite.LoadDesignLayout property and TLMDDockPersister.ApplyDesignLayout method allows to defer applying design-time layout in application to prevent flickering.

To get a quick start we suggest that you check the demo projects which are shipped with any version of LMD DockingPack (subfolder \lmddocking in \demos directory):

Description
\Docking Demo Demonstrates use of DockingPack controls

The new Visual Theme Editor can be found in the \tools menu of the installation directory.

RichPack Info
Welcome to LMD RichPack! This package contains several controls which are based on MS RichEdit DLL (including support for version 3) and which allow integration of text editors providing predefined dialogs (like Preview dialog with Zoom features) and hypertext functionality. In addition the package contains resource saving label controls which can display RTF text including hyperlinks and graphics.

Important Note for C++ Builder user
 The Generate Toolbar features at designtime are not available in CBuilder 5.
After recompiling LMD RichPack packages, you might receive errors regarding CHARFORMAT2A types if you want to start an application using TLMDRichEdit components. Workaround: Simply enter in the header files Richedit:: before CHARFORMAT2A types (where compiler requires this, usually about three times). Example:
Change line...
void __fastcall GetAttributes(CHARFORMAT2A &Format);     
..to...
void __fastcall GetAttributes(Richedit::CHARFORMAT2A &Format);

SearchPack
Welcome to LMD SearchPack! This collection of components helps you adding advanced search features (complex search expressions including support for operators and wildcards) to your applications.

Important Note for C++ Builder user
 If you receive error E2019 in LMDREFINEParser.hpp when compiling a project:
Simply copy the LMDRefineParser.hpp Header file from \Resources\CBuilder_Headers directory into your SearchPack library path or replace manually line 38:

System::ShortString yyShortString; 
with
char yyShortString[255];
Note: This problem will occur only when you compile SearchPack packages yourself or the CBuilder compiler had modified header files.

ShellPack
Welcome to LMD ShellPack! The package contains shell controls which help you in various ways to create Windows Explorer like projects. Other included components provide access to undocumented windows dialogs or advanced system functions. Further more many new properties/methods and especially TAction-objects were added to simplify work with Shell items. Major changes in version 4 are tight integration into new LMD VCL infrastructure, the new TLMDShellConsoleView component and other enhancements.

Important Note for C++ Builder user
If your application uses ShellPack components, the following line must be added in one of the units which contain ShellPack components (e.g. main unit):

#include <LMDShellBCBSupport.hpp>
If this header file is not included, an E2076 linker error will be generated.
