Installing the InfoPower help file

You can install InfoPower 4000’s online help by running the OpenHelp utility. To can do this, by clicking on Help | Customize from the Delphi IDE.  From there, click on Edit | Add Files to add ip4000d7.hlp (Delphi 7), ip4000c5.hlp (Builder 5), ip4000d6.hlp (Delphi 6) or ip4000c6.hlp (Builder 6) to the list of help files in the index and link tab pages. Do not add the help file to the contents tab page. After adding the InfoPower help file, click on File | Save Project to save your changes.

 

Warning – Delphi’s default help indexing is near the limits of Microsoft’s WinHelp.  As a result, you may need to remove any indexes you do not regularly use do stay within the capabilities of WinHelp.  If you experience an empty list on the Index tab page of WinHelp, this indicates you need to remove some .hlp files using OpenHelp.

 

Demo Applications

For an overview of the IP functionality, we suggest you compile the main demonstration program for InfoPower 4000. This demo is located at \ip4000\demos\maindemo\prjdemo.dpr.  At this time the demos have not been updated for InfoPower 4000’s new functionality.

 

Note to Delphi users who are using the InfoPower 4000 TwwDBRichEditMSWord component:

It is important that you install the correlating InfoPower RichEditMSWord package based on the Office automation package you have selected.  For instance, if you install the office xp automation package (Dclofficexp70.bpl) you should use the IP4000WORDXPVCL7 design-time package, instead of IP4000WORDVCL5 or IP4000WORDVCL6. These packages are located in your \ip4000\package directory. You will also need change your uses clause in your Pascal source so that it references wwrichedspellxp, instead of wwrichedspell2000 or wwrichedspell.

 

We also recommend omitting the InfoPower IP4000WORD* runtime packages from your list of runtime packages for your project, as this removes the requirement to distribute both the corresponding ip4000word* packages, as well as the office automation package. Omitting the these InfoPower runtime packages from your project will add about 10K to your executable size.

 

C++ Builder 5 and 6 notes: The TwwDBRichEditMSWord component is not supported in C++ Builder
